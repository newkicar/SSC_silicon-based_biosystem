# 上下文交接（Context Handoff）

> 借鉴 Sliver Vibe Coding 的上下文交接思想，当对话上下文已满、或接手他人项目时，生成结构化的交接文本。

---

## 何时使用

以下场景需要生成交接文本：

1. **对话上下文已满**：AI 响应变慢、遗忘早期对话内容
2. **接手他人项目**：从另一个 AI 或另一个 session 接手
3. **换窗口继续**：用户想在新的 Cline 窗口继续工作
4. **换 AI 助手**：从一个 AI 工具切换到另一个

---

## 交接文本格式

```markdown
## 项目交接

### 1. 项目概况
- **项目描述**：[一句话描述]
- **当前阶段**：[立项/开发中/验收中/发布中]
- **最近完成**：[最近完成的主要事项]

### 2. Git 状态
- **当前分支**：[branch-name]
- **最近提交**：[commit hash + message]
- **未提交改动**：[是/否，如有请列出]

### 3. 验证证据
- **测试**：[pytest/npm test 结果]
- **Lint**：[ruff/eslint 结果]
- **CI**：[最新 CI 状态]

### 4. 已知风险
- [风险1：描述 + 缓解措施]
- [风险2：描述 + 缓解措施]

### 5. 下一步命令
- [下一个应该执行的命令或工作流]
```

---

## 自动生成

在 session-end Phase 6 中，如果检测到以下情况，自动生成交接文本：

1. `memory/progress.md` 中有未完成的 Next 项
2. `memory/blockers.md` 中有活跃 blocker
3. CI 状态为失败
4. 对话长度超过 100 轮

---

## 使用方式

**手动触发**：
```
给我一份交接文本，我要换窗口继续。
```

**自动触发**：
- 对话上下文即将溢出时
- 检测到重复 AI 修复循环时
- 任务被 blocker 卡住超过 3 轮时
