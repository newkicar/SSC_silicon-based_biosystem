# 自然语言路由（Natural Language Routing）

> 借鉴 Sliver Vibe Coding 的自然语言自动路由思想，让用户不需要记命令，直接说需求即可。

---

## 核心思想

**用户不需要记 slash command**。AI 根据用户的自然语言描述，自动判断应该使用哪个工作流。

---

## 路由映射表

| 用户说 | AI 自动路由到 |
|--------|-------------|
| "我想从零做一个项目" | `/立项` → new-feature-workflow |
| "我接手了一个写了一半的项目" | `/接管项目` → 只读审计 |
| "项目跑不起来" | `/环境启动` → 建立启动基线 |
| "帮我选技术栈" | `/技术选型` → 推荐唯一路线 |
| "我要加一个功能" | `/功能开工评估` → 风险评估 |
| "我要开始做这个功能" | 检查真源 → `/执行子阶段` |
| "没有实施真源，可以直接开发吗" | 阻止 → 先补真源 |
| "要接 API、SDK、OAuth、支付" | `/第三方接入` → 查官方文档 |
| "没有 git / 内部资料不能推" | `/git保护` → 建立保护规则 |
| "我要回滚 / revert 这个 commit" | `/git保护` → 损失审计 |
| "上下文太长，换窗口继续" | `/上下文交接` → 生成交接文本 |
| "收工" | session-end → 验证 + Memory 更新 |
| "用 speckit 流程" | speckit 子体系 |
| "修这个 bug" | bugfix-workflow |
| "项目越来越乱了" | `/AI债务体检` |
| "报错了，一直修不好" | `/报错救援` |
| "上线" / "部署" | `/部署路线` → `/发布准备` |
| "我怎么确认" / "我不会验收" | `/用户验收陪跑` |
| "感觉跑偏了" | `/防漂移` |

---

## 执行流程

```
用户说自然语言
    │
    ▼
AI 判断意图
    │
    ▼
风险评估（参考 task-risk-gates.md）
    │
    ├── 普通工程任务 → 窄查、窄改、窄验
    ├── 轻量任务 → scoped read-edit-validate
    ├── 标准任务 → 需要真源 + 用户确认
    └── 高风险任务 → 强制完整治理
    │
    ▼
选择对应工作流
    │
    ▼
执行 + 验证
```

---

## 与现有工作流的关系

| 原有方式 | 新增方式 |
|---------|---------|
| 用户说「收工」 | AI 自动判断任务完成，建议收工 |
| 用户说「用 speckit 流程」 | 用户说「从零定义需求」时自动推荐 |
| 用户说「小修 bug」 | 用户说「修个 bug」时自动路由到 bugfix-workflow |
