# 用户验收陪跑（User Acceptance Walkthrough）

> 借鉴 Sliver Vibe Coding 的用户验收陪跑思想，把技术验证变成非技术用户能理解的点击路径、预期界面、通过/失败判断。

---

## 适用场景

以下场景需要使用用户验收陪跑：

1. **非技术用户验收**：用户不懂技术，需要把测试变成点击路径
2. **功能验收**：新功能开发完成后，用户需要确认功能是否正确
3. **上线前验收**：准备上线前，需要用户确认所有功能正常

---

## 验收清单格式

```markdown
## 功能验收清单

### 功能：[功能名称]
**验收人**：[用户姓名]
**验收日期**：[YYYY-MM-DD]

---

### 步骤 1：[步骤描述]
- **操作**：点击 [按钮名称]
- **预期结果**：看到 [页面/消息]
- **通过标准**：[具体判断标准]
- **实际结果**：[ ]
- **备注**：

### 步骤 2：[步骤描述]
- **操作**：输入 [内容]，点击 [按钮名称]
- **预期结果**：看到 [页面/消息]
- **通过标准**：[具体判断标准]
- **实际结果**：[ ]
- **备注**：

### 步骤 3：[步骤描述]
...

---

### 验收结论
- [ ] 全部通过
- [ ] 部分通过（见备注）
- [ ] 未通过（见备注）

### 备注
[用户填写]
```

## 技术验证 ↔ 用户验收映射

| 技术验证 | 用户验收 |
|---------|---------|
| `pytest test_login.py` | 输入用户名和密码，点击登录，看到欢迎消息 |
| `ruff check .` | 页面没有报错信息，布局正常 |
| `npm test` | 所有功能按钮都能正常点击 |
| CI 通过 | 网站能正常访问，没有 500 错误 |
| 数据库迁移 | 新用户能正常注册，老用户数据没丢 |

---

## 自动生成

在 `verify-changes.md` 验证通过后，自动生成用户验收清单：

1. 读取 `specs/acceptance-criteria.md` 中的验收标准
2. 将技术语言翻译成用户语言
3. 生成可执行的点击路径
4. 输出给用户确认
