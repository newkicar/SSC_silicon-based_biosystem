# PRD（产品需求文档）

> 本文档针对 HR SSC 硅基生物系统（组织数字孪生）的整体产品需求。

---

## 背景

将 HR SSC（人力资源共享服务中心）重构为一个**以 AI 为神经中枢的硅基生命体**——能感知、能思考、能进化、能与人类协同。

当前痛点：
- HR SSC 员工需手动处理大量重复性咨询（政策、考勤、花名册、加班数据等）
- 数据分散在多个 Excel/数据库，缺乏统一检索和分析能力
- 管理层缺乏数据驱动的洞察通知，依赖人工汇总报表
- 跨部门任务分派效率低，缺乏自动化编排能力

本项目通过 deepagents + LangChain 构建 AI 原生 HR 系统，实现：
1. 员工自助问答（政策/考勤/花名册查询）
2. 多步骤业务流程自动执行（Marathon 引擎）
3. 数据洞察自动生成与精准推送
4. RBAC 权限控制的精细化角色管理

## 目标用户

| 用户群体 | 数量 | 说明 |
|---------|------|------|
| 员工终端（CLI） | 34 | 通过 Staff 终端交互，查询个人数据 |
| SSC 操作层 | 9 | HR SSC 员工，处理日常事务 |
| 管理层 | 34 | 接收数据洞察通知，辅助决策 |
| 管理员 | 1 | 系统配置与维护 |

## 功能范围

### 必须做（In Scope）

1. **智能问答系统**
   - Web 门户（RAG 政策搜索）
   - CLI 终端（联合搜索：RAG + 数据库）
   - 身份识别与渠道分流

2. **Marathon 多步骤执行引擎**
   - LLM 自动拆解复杂任务为子步骤
   - 按 capability 动态注入工具集
   - RubricMiddleware 评分自校正
   - execute_skill 幂等性缓存

3. **数据洞察系统**
   - 每日自动数据更新（花名册 + 加班数据）
   - 洞察子代理自动生成分析
   - 精准通知路由（按角色/部门/公司隔离）

4. **RBAC 权限体系**
   - 19 个预定义角色
   - 排除法权限控制
   - 兼岗并集支持

5. **定时任务调度**
   - 数据更新、洞察生成、记忆整理、心跳巡检

### 不做（Out of Scope）

1. **SAP 系统直连**：仅通过 Excel 文件间接读取，不直接对接 [ERP系统接口]
2. **移动端 App**：仅支持 Web 门户和 CLI 终端
3. **语音/视频交互**：当前仅支持文本
4. **外部邮件系统集成**：通过 execute_skill 调用本地 Outlook 工具间接实现

## 非功能需求

- **性能**：单次问答响应 ≤ 5s（不含工具调用），Marathon 复杂任务 ≤ 60s
- **安全**：OAuth2 认证 + RBAC 权限，敏感数据按 scope 隔离
- **可靠性**：定时任务失败自动创建 HRIS 紧急工单
- **可扩展性**：新增数据源只需放入 `databases/` 目录，重启即自动索引
- **可观测性**：结构化日志 + Dashboard KPI 监控
